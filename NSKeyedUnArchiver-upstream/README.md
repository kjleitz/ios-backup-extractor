# NSKeyedUnArchiver

Unserializes any binary|text|file|memory plist data and returns a usable Python dict.

Plist files are very common in Apple platforms (macOS, iOS etc). Plist files are used to store data much like XML and JSON files, but can be pure text or binary (convertible with plutil command). Some Plist files were generated by Apple NSKeyedArchiver and contain a serialized version of objects stored in an intricate and complex schema of nested objects. NSKeyedUnArchiver converts it back to an hierarchical form as a Python dict.

Uses only Python 3.8 plistlib, no other dependencies.

Automatically converts well known data types to Python-equivalent data types:

- `NSArray` ➔ `list`
- `NSMutableDictionary`, `NSDictionary` ➔ `dict`
- `NSMutableString`, `NSString` ➔ unwrap the `string`
- `NSMutableData`, `NSData` ➔ unwrap the data
- `NSDate` ➔ `datetime`

Check [pypi](https://pypi.org/project/NSKeyedUnArchiver/) for packages.

See package name for your system package manager at [Repology](https://repology.org/project/python:nskeyedunarchiver/information).

License is LGPL 3.

## Sample usage

```python
import NSKeyedUnArchiverLocal

file='/path/to/file.plist'
my_dict=NSKeyedUnArchiverLocal.unserializeNSKeyedArchiver(file)


data=b'bplist00\xd4\x01\.........'
my_dict=NSKeyedUnArchiverLocal.unserializeNSKeyedArchiver(data)


data='<plist version="1.0">......'
my_dict=NSKeyedUnArchiverLocal.unserializeNSKeyedArchiver(data)


file=PurePath('/path/to/file.plist')
my_dict=NSKeyedUnArchiverLocal.unserializeNSKeyedArchiver(file)
```

## This fork (`NSKeyedUnArchiver-local`)

This is a locally patched fork of the upstream [NSKeyedUnArchiver](https://github.com/avibrazil/NSKeyedUnArchiver) package. It is published as `NSKeyedUnArchiver-local` to avoid conflicting with the upstream PyPI package.

### Changes from upstream

- **Package renamed** — module is now `NSKeyedUnArchiverLocal` (install: `NSKeyedUnArchiver-local`).

- **Fix bare `except:` clause** — The inner `try/except` in `_unserialize` that handles nested binary plist data used a bare `except:`, which silently swallowed `KeyboardInterrupt`, `SystemExit`, and other non-`Exception` signals. Replaced with `except (plistlib.InvalidFileException, ValueError, KeyError):`.

- **Replace `print()` debug statement with logger** — An unconditional `print()` call dumped arbitrary plist values (potentially sensitive) to stdout. Replaced with `module_logger.debug(...)` so output is gated behind Python's standard logging system.

- **Fix `finished` flag logic bug in `_unserialize`** — Inside the `for k in cursor` loop, a `finished=True` statement at the end of the `elif dict/list` branch would reset the UID-resolution flag set by an earlier key in the same pass. This caused the `while not finished` loop to exit prematurely, leaving unresolved `plistlib.UID` objects in the output for some inputs. The erroneous `finished=True` was removed; the assignment at the top of the `while` loop is sufficient.
